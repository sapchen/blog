# 《家庭维修手册》背后的故事
## 初心
&emsp;&emsp;这本书最早是在微信公众号上面连载的，一开始做的很简单，就是记录自己维修自行车的过程。起因是去年初的经济下行，单位揭不开锅了，家里老婆又怀孕了，真是“屋漏偏逢连夜雨”。当时想的很清楚，自己骑自行车将近10年了，因为维修昂贵，都是自己动手操作，那如果工作保不住了可以试一下去迪卡侬应聘自行车修理工。去年上半年的情况大家也都经历过，美联储加息丝毫没有停下的脚步，全球为美帝买单。于是在此基础之上，我又在公众号上分享了自己剪头发的技术实现细节，还有一些别的DIY技术经验分享，当时的想法很单纯，这是全国的问题，整个大环境不好，那我就把自己最最精华（压箱底）的技能分享出来，帮助大家渡过这个难关。同时，通过运营公众号，也能通过实践深刻理解流量，以及其给账号主理人带来的好与坏两方面的影响，做大是一个目的，更重要的是通过运营公众号实现自我的修行，这是中国传统文化教育我们的。因此，在完成了修行之后，我主动注销了自己一度月流量达到60W的微信官方认证生活类自媒体公众号。

## 维修家
&emsp;&emsp;既然本来想的是维修自行车，为什么后续又连载“维修家”主题的内容呢？这就是时机上的巧合了，23年年底（11月），我和宸宸在办完婚礼喜酒之后，很自然地搬入了这个我们俩亲自设计和布局的新房子，我们也自己开灶了，然后我们就发现了很多装修中遗留下来的问题。比如厕所湿区的玻璃推拉门总是发出短暂刺耳的噪声，客厅的风管机不制冷，房间的木质衣橱推拉门不正导致抽屉拉不出来，厨房的洗菜池有臭味。等等一系列问题叠加上一段落已经描述的事业和家庭的双重压力，直接让我开挂暴走了，我决定亲自解决遇到的所有问题，舍弃掉不重要的，保留最看重的，最后把这个小家梳理好。于是，我把微信里面记录下来的所有维修电子设备等等图片全部整理成公众号文章一并做了分享。

## honkit电子书
&emsp;&emsp;那公众号连载就连载了，为什么我又要做成honkit电子书发布在github上面呢？这其实多亏了deepseek，我发现整个公众号的流量非常小（平均一篇不到100阅读量），感觉这种零散的分享似乎得不到广泛的传播（此刻时间已经来到了今年的8月，距离去年2月开通公众号已经1年半）。因此在和deepseek的一次畅聊中，deepseek建议我整理成册，部署在github pages上面，我觉得是个好主意，公众号上的文章太乱太杂，即时做了主题的分类，还是不够有序。因此我最早计划的是用gitbook做成开源书将其公开发布。后来随着项目进入实施阶段，我发现gitbook的技术已经濒临淘汰，现在流行的是honkit（deepseek告诉我他是gitbook的一个社区分支），于是整个技术方案变成了honkit + github pages，本来我是想发布在gitee的pages上面，后来发现gitee对个人已经关闭pages的使用权限了，就没办法，选择了github pages发布。随着第一章节内容的完成，deepseek又推荐我使用github actions全自动部署，可以减轻本地部署的代码量，这下整个技术路线就很清晰了：honkit + github pages + actions。于是，这本电子书就逐步逐步诞生了。

## 关于引用母校的风景图
&emsp;&emsp;这真的是一个意外，一开始我在完成了第一章节维修自行车的内容之后，还没有上线github。我用honkit在本地生成了epub电子书后，发到了学校的“梦想单车”自行车群，发现效果出乎意料地好，我有一些震惊，于是在内心确认了发布pages的目标。在逐步整理完成了前6章之后（排版配色等等尚未完善），我上线github pages，并尝试写公众号文章、发微博等方式进行推广，但是我发现流量非常小，很久很久公众号总阅读量不到30，微博阅读量更是只有个位数。所以，最后，我试着把链接发到母校的论坛，想的很清楚，我的计算机技术最初来源于这里（linux技术），现在是我回馈的时候了。我将链接发到学习交流区，马上就被引爆，连着上了两天“Top 10”，github上面的repo star数持续增长，直到突破至50个star（我自己不算在内）。至此，我明白了两件事，第一，校友自带强信任关系，第二，这本书真的有价值。在学弟学妹们的关注下，电子书内容一步步完善，第七章“升级汽车”也被逐步开发出来。做为回馈，我也把这本书作为母校70周年校庆的礼物在论坛学习交流区进行了说明，因此就很自然地配上了母校的风景照，用来感谢学弟学妹们的支持和关心。

## 本文最后
&emsp;&emsp;这个项目的起源公众号已经被我注销了，因为后期随着升级汽车和烧菜内容的发布，流量巨大，我也遭受到了流量的反噬，身体上出了问题（失眠等等），最终我转向轮现在的工作模式，以github为核心，再单向精选内容至全平台发布，再由全平台单向引流至github基地。于是我重新注册了同名未认证微信公众号，新的由内至外的个人成长模式已经开启，欢迎大家关注！

&emsp;&emsp;回见，各位githubers！
